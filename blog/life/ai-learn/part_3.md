
# 关键词解释

你列出的这些关键词，分散在此前我们讨论的几篇不同论文里，但都围绕着 **LLM 智能体的构建、攻击与防御** 这一核心脉络。

为了方便你理解，我将它们按照所属的论文和内在逻辑，分为四个主题来解释。

---

### 📝 第一部分：ReAct 论文（推理与行动协同）核心术语

这一部分主要对比 ReAct 与其之前技术的区别，并介绍其关键概念。

- **思维链 (Chain-of-Thought, CoT)**
    
    - **角色**：ReAct 论文中对比的主要基线。
        
    - 概念：早期是从问题到答案的直接映射，现在在给出答案之前分步骤推导，其模型会在在训练过程中在训练数据中放入推导模式，初期在用户输入提示词的时候激活模型的模式匹配推导能力，现在已经默认是输入提示词之后，自动生成推导过程，然后分步骤解答。
        
    - **解释**：一种提示方法。让模型在给出最终答案前，先输出一段“思考过程”。其**主要缺陷**是推理过程是“静态且封闭的”，仅依赖模型内部知识，容易产生事实幻觉和错误传播。
        
- **纯行动模型 (Act-only)**
    
    - **角色**：ReAct 论文中对比的另一个基线。
        
    - **解释**：指只有“行动”能力的模型。它直接根据输入决定操作，缺乏规划和反思，显得“盲目且短视”，在复杂长程任务中容易迷失。
        
- **(Inner Monologue, IM)**
    
    - **角色**：ReAct 论文中对比的工作，用于论证“主动思考”的价值。
        
    - **解释**：一种早期的具身智能体方法。其“内心独白”仅仅是对环境状态或未完成目标的简单**复述**，非常“被动且肤浅”。ReAct 的进步在于“主动思考”。
        
- **纯行动控制基线**
    
    - **解释**：即上述的“纯行动模型”，在实验中作为被严格控制变量的对照基准。
        
- **稀疏与密集推理**
    
    - **解释**：ReAct 中推理发生的频率。
        
        - **密集推理**：在知识密集型任务（如多跳问答）中，几乎每一步行动前后都需要思考。
            
        - **稀疏推理**：在决策任务（如游戏）中，思考只出现在关键节点（规划、遇阻时），避免中断流畅动作。
            
- **轨迹 (Trajectory)**
    
    - **解释**：模型求解一个任务的完整过程记录，即 `Thought 1 -> Action 1 -> Observation 1 -> Thought 2 -> ...` 的序列。在上下文学习中，这些作为“示例”教模型如何交替思考与行动。
        
- **假阳性 (False Positive)**
    
    - **解释**：在论文语境下，指模型看似推理成功（产生了完整的推理链），但给出的**最终答案却是错误的**。CoT 的“假阳性”率远高于 ReAct。
        
- **自洽性基线 (CoT-SC)**
    
    - **解释**：一种增强方法，即让 CoT 以稍高**解码温度**生成多条推理路径，取“多数表决”的结果。**解码温度**越高，模型输出越多样。
        
- **自举方法 (Bootstrapping)**
    
    - **解释**：ReAct 微调时用到的策略。先用大模型生成高质量正确答案和轨迹，再用这些数据去**微调**小模型，让后者学会推理和行动。
        
- **微调扩展效应 (Scaling Effect of Finetuning)**
    
    - **解释**：指不同模型在不同情况下，通过微调获得的性能提升程度。
        
- **高度隐式**
    
    - **解释**：指输入到输出的映射关系极其复杂，无法用简单规则描述，需要大量计算或深层推理。
        
- **语言先验 (Language Priors)**
    
    - **解释**：模型在海量文本训练中学到的语言知识、常识和推理模式，是 ReAct 能产生合理“思考”的基础。
        
- **多模态观测与具身任务**
    
    - **解释**：**多模态观测**指环境给出的不仅是文本，还有图像、声音等；**具身任务**指机器人在物理世界中完成的真实操作。ReAct 论文里主要处理的是文本任务。
        
- **稀疏奖励 (Sparse Reward)**
    
    - **解释**：智能体只在完成最终目标时获得奖励，中途几乎没有反馈，会导致学习困难。ReAct 的自我推理能力为智能体提供了内在指引。
        
- **可供性模型 (Affordance Model)**
    
    - **解释**：在机器人决策中，该模型用于判断“在当前环境下，能做什么动作”。ReAct 通过逻辑推理来替代这一功能。
        
- **视觉环境接地 (Grounded on the Visual Environments)**
    
    - **解释**：“接地”指将抽象符号与真实世界的感知经验联系起来。这里特指机器人的决策建立在视觉输入上，而非纯文本。

### 🛡️ 第二部分：OWASP 代理安全框架关键术语

此部分区分了 AI 代理安全中的两个新概念。

- **隐式耦合 vs. 显式信任**
    
    - **解释**：
        
        - **隐式耦合**：代理之间默认“你相信我，我也相信你”，只要在内部网络就自动信任，是巨大的安全隐患。
            
        - **显式信任**：代理间必须主动出示身份凭证（如令牌），通过验证后才能建立信任关系。**基于声明的多因子认证**是实现方式之一，要求代理提供多重身份证明材料。

### 🎯 第三部分：ToolHijacker 攻击论文关键术语

此部分聚焦针对工具选择过程的攻击。

- **召回 (Recall)**
    
    - **解释**：攻击第一步。攻击者要确保其**恶意工具文档**在检索阶段，能被检索模型成功找到并放入候选列表。
        
- **形式化与两阶段优化**
    
    - **解释**：“形式化”指将攻击构建成数学优化问题。“两阶段优化”指先确保被**检索**到，再确保被 LLM **选择**。
        
- **无盒优化目标**
    
    - **解释**：攻击设定为“无盒”，即没有任何内部反馈信号。攻击者只能在本地，用公开模型和自己的目标来模拟优化，以生成有效攻击文本。
        
- **选择器 (Selector)**
    
    - **解释**：工具选择流程的第二阶段，通常是 LLM 本身，从候选工具中最终选择一个。
        
- **通用嵌入模型 (General-purpose Embedding Model)**
    
    - **解释**：攻击者用来近似**语义相似度**计算的开源模型，以模拟受害者检索器。**语义相似度**用于衡量文本间含义的相近程度。
        
- **受害者模型 (Victim Model)**
    
    - **解释**：被攻击的目标系统。
        
- **烧蚀实验 (Ablation Study)**
    
    - **解释**：一种分析方法，通过移除系统组件来验证各部分的贡献。ToolHijacker 的烧蚀实验证明，其两阶段策略缺一不可。

### 🧠 第四部分：WebInject 攻击论文关键术语

此部分聚焦利用对抗样本的视觉注入攻击。

- **对抗样本 (Adversarial Example)**
    
    - **解释**：在输入上添加人眼不可察的微小**扰动**，导致模型出错。
        
- **神经网络代理 (Neural Network Proxy)**
    
    - **解释**：WebInject 的核心创新。为解决浏览器渲染管线（**黑箱**）不可微的问题，用一个**可微的神经网络**去模拟该过程，从而能用**投影梯度下降法**计算扰动。攻击在一个“**灰盒**”场景下进行（知道模型架构，但不直接控制输入）。
        
- **投影梯度下降法 (Projected Gradient Descent, PGD)**
    
    - **解释**：用于生成对抗样本的算法。核心迭代步骤是：先沿**负梯度**方向添加扰动，再将扰动限制在一定范围内。
        
- **U-Net**
    
    - **解释**：一种“编码器-解码器”架构的神经网络，常用于图像生成任务，被 WebInject 用作代理网络。
        
- **视觉对抗攻击 (VisualAdy)**
    
    - **解释**：泛指在视觉领域生成对抗样本的方法，相当于“普通攻击”。
        
- **迁移攻击 (Transfer Attack)**
    
    - **解释**：利用深度学习迁移性，用A模型生成对抗样本直接攻击B模型，即“用假想敌练的招，去打真正对手”。
