不需要自动序列化到了，太麻烦了，不利于追踪类型问题和自动化工具的识别。


1、生成布局接口使用 /api/models/generate

请求类型是 

export interface GenerateModelRequest {

    force_regenerate?: boolean;

    layout_id?: string;

    response_mode?: string;

    [property: string]: any;

}

响应为

export interface GenerateModelResponse {

    message?: string;

    status?: number;

    [property: string]: any;

}

2、生成进度接口 /api/models/progress/{layoutId}

响应为 

export interface Response {

    message?: string;

    status?: number;

    [property: string]: any;

}

状态 10 对应启动
80对应生成之中
100对应生成结束

3、 结束后使用接口获取模型 /api/models/result/{layoutId}


响应是

export interface Response {

    models?: ModelFileInfo[];

    status?: number;

    [property: string]: any;

}



export interface ModelFileInfo {

    modelId?: string;

    objContent?: string;

    objPath?: string;

    [property: string]: any;

}




这是获取模型列表信息接口 /api/models/list


响应结构

export interface Response { layout_with_models_list?: LayoutWithModels[]; message?: string; [property: string]: any; } 

export interface LayoutWithModels { layout?: LayoutsListItem; models?: ModelListResponseDTO[]; [property: string]: any; } 

export interface LayoutsListItem {  layout_id?: string;layout_name?: string; status?: number; svg_url?: string; [property: string]: any; } 

export interface ModelListResponseDTO { model_id?: string; model_name?: string; status?: number; [property: string]: any; }

然后在下拉菜单选择不同布局的时候根据不同布局下的多个的模型id获取实际模型文件进行渲染


接口为 /api/models/model/{modelId} 这是响应


export interface Response { models?: ModelFileInfo[]; status?: number; [property: string]: any; } 
export interface ModelFileInfo { modelId?: string; objContent?: string; objPath?: string; [property: string]: any; }
