
一、	课题背景及意义（含国内外研究现状综述）

**课题背景**

随着计算机图形学、虚拟现实（VR）、增强现实（AR）以及元宇宙概念的兴起，三维模型作为数字化世界的核心资产，其需求正在爆炸式增长。然而，传统的三维建模过程通常依赖于专业软件（如 Blender、3ds Max）和具备专业技能的设计师，这一过程不仅耗时耗力，且技术门槛高，限制了普通用户进行三维内容创作的普及性。

近年来，人工智能生成内容（AIGC）技术取得了突破性进展，尤其是在文本到图像生成领域。将这种能力扩展到三维模型生成，即通过自然语言描述直接合成三维模型，已成为学术界和工业界的研究热点。这不仅能极大降低三维内容创作的门槛，还能为游戏开发、影视制作、工业设计、数字孪生等领域带来革命性的效率提升。

**国内外研究现状综述**

当前，三维模型生成技术主要分为几条路线：

1.  **基于优化/渲染的方法**：这类方法通常利用预训练的视觉-语言模型（如 CLIP）作为引导，通过可微分渲染器迭代优化三维模型的几何形状（如网格、点云或神经辐射场 NeRF）和纹理，使其渲染后的二维视图与输入文本描述的语义保持一致。例如，DreamFusion [1] 和 Magic3D [2] 利用强大的二维扩散模型作为先验知识，生成了高质量的三维模型，但其优化过程通常非常耗时。

2.  **直接生成方法**：这类方法尝试训练一个深度学习模型直接输出三维模型的表示。例如，一些工作使用生成对抗网络（GANs）或自回归模型来生成体素（Voxel）或点云表示，但往往受限于分辨率和拓扑结构的复杂性。

3.  **基于大语言模型（LLMs）的方法**：随着 LLMs 在代码生成和结构化数据理解方面展现出强大能力，研究人员开始探索利用 LLMs 直接生成描述三维模型的声明式代码（如 OpenSCAD、SDF、Three.js 代码等）。这种方法的优势在于能够生成结构清晰、可编辑、参数化的模型。然而，LLMs 在处理连续的坐标数值和复杂的空间几何关系时仍面临挑战，容易产生“幻觉”或语法错误，导致模型无法正确渲染 [3]。正如 LLM4SVG [4] 在二维矢量图形领域指出的，LLMs 将 SVG 源码视为普通文本，忽略了其分层和结构化特性，导致生成质量受限。类似地，在三维领域，直接让 LLM 生成复杂的网格数据或场景描述文件也存在同样的问题。

综上所述，利用大语言模型直接生成结构化、可编辑的三维模型是一个充满前景但挑战重重的方向。本课题旨在借鉴二维矢量图形生成领域的先进思想，探索一种新的方法，使大语言模型能够更深入地理解三维几何结构和外观属性，从而生成高质量、可交互的三维模型。

二、	课题研究主要内容及研究基础

**课题研究主要内容**

本课题旨在设计并实现一个基于大语言模型（LLM）的三维模型生成及可视化交互系统。核心研究内容包括：

1.  **三维模型的语义化表示研究**：借鉴 LLM4SVG [4] 对 SVG 的处理方式，本研究将探索一种针对三维模型的结构化、语义化表示方法。我们将定义一系列“语义标记”来表示三维基本图元（如立方体、球体、圆柱体）、变换操作（如平移、旋转、缩放）以及材质和颜色属性，替代原始的、冗长的代码或数据格式。这有助于 LLM 更好地理解三维模型的构成，减少生成过程中的语义模糊。

2.  **面向三维模型生成的指令微调数据集构建**：当前缺乏大规模、高质量的“文本描述-三维模型”指令数据集。本研究将设计一个自动化数据生成流水线，收集并处理现有的三维模型库（如 ShapeNet、Thingi10k），为模型生成对应的文本描述和结构化的指令数据，构建一个包含数十万量级的指令微调数据集。

3.  **集成几何与语言特征的 LLM 架构设计**：在现有 LLM（如 LLaMA、Phi-3 等）基础上，设计一个模块化的增强架构。该架构将集成一个用于编码三维指令的编码器和用于精确预测坐标、尺寸和颜色等连续值的回归头。目标是让模型能够有效融合几何、外观和语言信息，生成语义一致且几何精确的三维模型。

4.  **三维模型可视化与交互系统的实现**：基于 WebGL 和 Three.js 等前端技术，开发一个可视化交互系统。该系统能够接收并解析 LLM 生成的三维模型代码，在网页端进行实时渲染。同时，系统将提供基础的交互功能，如视角控制（旋转、缩放、平移）和对生成模型的简单编辑，以验证生成模型的可用性和可编辑性。

**研究基础**

本研究将基于现有成熟的开源技术和框架。指导教师在相关领域有深入研究，实验室具备所需的计算资源。我已掌握 Python 编程、深度学习基础，并对 PyTorch、Transformers 等框架有一定了解。通过前期文献调研，对三维模型生成的主流技术路线和 LLM 微调技术有了初步认识，为本课题的顺利开展奠定了基础。

三、	研究（或调研）方案和思路（技术路线）

本课题的技术路线如下图所示，主要分为数据准备、模型训练和系统实现三个阶段。

1.  **数据准备阶段**：
    *   **数据收集**：从开源三维模型网站（如 Thingiverse）和学术数据集（如 ShapeNet）收集大量三维模型（如 `.obj`, `.stl` 格式）。
    *   **数据预处理**：编写脚本对模型进行规范化处理，如统一尺寸、简化网格、修复破损等。
    *   **指令数据生成**：将规范化后的三维模型转换为一种中间结构化表示。利用多模态大语言模型（如 LLaVA、GPT-4V）为模型生成丰富的文本描述。基于此，结合预定义的指令模板，自动生成大规模的“指令-响应”数据对。

2.  **模型训练阶段**：
    *   **模型选型**：选择一个合适的开源大语言模型作为基础模型，如 Phi-3 或 Qwen2。
    *   **语义标记化**：扩展 LLM 的词汇表，加入代表三维操作的“语义标记”，并设计初始化方法（如使用文本描述的嵌入向量进行初始化）[4]。
    *   **两阶段训练策略**：
        *   **第一阶段：特征对齐预训练**。冻结 LLM 主体参数，仅训练新加入的语义标记嵌入层和投影层，使模型初步对齐三维模型的结构化表示。
        *   **第二阶段：端到端全量微调**。使用 LoRA 等参数高效微调技术或全量微调，在完整的指令数据集上训练整个模型，增强其遵循指令生成三维模型的能力。

3.  **系统实现与评估阶段**：
    *   **后端开发**：使用 Python (Flask/FastAPI) 部署微调后的 LLM，提供 API 接口。
    *   **前端开发**：使用 HTML/CSS/JavaScript 和 Three.js 库构建前端界面。用户输入文本指令，前端调用后端 API 获取生成的模型代码，并由 Three.js 解析渲染。
    *   **评估**：通过定量（如与原始模型的几何相似度评估）和定性（如人工评估生成模型的视觉质量和文本对齐度）方法，对模型性能进行综合评估。
