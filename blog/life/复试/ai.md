## 基本参数

下面是你代码中各个参数的含义解释：

1. **--dataset**  
   - 含义：指定要使用的数据集配置文件的路径（通常是一个 .yaml 文件，里面描述了训练/验证图片和标签的路径等信息）。
   - 示例：`--dataset ./dataset2-feiyi.yaml`

2. **--weight**  
   - 含义：指定训练时加载的初始权重文件路径（如预训练模型的 .pt 文件）。可以用来迁移学习或断点续训。
   - 示例：`--weight ./yolo11n.pt`

3. **--epochs**  
   - 含义：训练的总轮数（epoch），即整个数据集会被训练多少遍。
   - 示例：`--epochs 3`

4. **--batch_size**  
   - 含义：每个训练批次（batch）包含的样本数量。影响显存占用和训练速度。
   - 示例：`--batch_size 4`

5. **--img_size**  
   - 含义：训练时输入图片的尺寸（通常为正方形边长）。一般不用改，除非有特殊需求。
   - 示例：`--img_size 640`

这些参数用于灵活控制模型训练过程，适应不同的数据集和硬件环境。

## yolo介绍

### YOLO11 是什么版本？

YOLO11 是 YOLO（You Only Look Once）系列计算机视觉模型的最新迭代版本。它在 YOLOv8 的基础上进行了多项重要改进：

核心改进

- **参数更少，精度更高**：YOLO11m 相比 YOLOv8m 参数减少了 **22%**，但在 COCO 数据集上实现了更高的平均精度（mAP）
- **速度更快**：推理时间比 YOLOv10 快约 **2%**，更适合实时应用
- **重新设计的架构**：改进了主干和颈部结构，增强了特征提取能力

### 模型规格
YOLO11 提供五种尺寸的模型：
| 型号              | 参数量   | 适用场景             |
| ----------------- | -------- | -------------------- |
| YOLO11n (Nano)    | 260万    | 边缘设备、轻量级任务 |
| YOLO11s (Small)   | -        | 平衡型应用           |
| YOLO11m (Medium)  | -        | 通用场景             |
| YOLO11l (Large)   | -        | 高精度需求           |
| YOLO11x (X-Large) | 约5600万 | 大规模应用           |

### 支持的任务
YOLO11 支持多种计算机视觉任务：
- 目标检测
- 实例分割
- 图像分类
- 姿态估计
- 旋转框检测（OBB）
- 目标跟踪

## 迁移学习和从头训练的区别

### 从头训练（Train from Scratch）
从头训练是指随机初始化模型权重，仅使用自己的数据集从零开始训练整个网络。

### 迁移学习（Transfer Learning）
迁移学习是指将预训练模型（已在大型数据集上训练好的模型）作为起点，针对自己的任务进行微调。预训练模型已经学会了通用的特征提取能力（如边缘、纹理、形状等），只需在新任务上调整少量层即可。

### 核心区别对比

| 对比维度     | 从头训练                               | 迁移学习                         |
| ------------ | -------------------------------------- | -------------------------------- |
| **数据需求** | 需要大量标注数据（通常数万张以上）     | 少量数据即可获得较好效果         |
| **训练时间** | 慢，需要完全收敛                       | 快，仅需微调少量层               |
| **计算资源** | 高，需完整训练过程                     | 低，大部分参数已收敛             |
| **模型效果** | 数据充足时可达到最优                   | 小数据集下效果明显优于从头训练   |
| **适用场景** | 任务特殊、无可用预训练模型、数据量极大 | 任务与预训练模型相关、数据量有限 |

### 什么时候选择哪种方式？

**选择从头训练**：
- 任务非常特殊，与现有预训练模型的训练数据分布完全不同
- 拥有海量标注数据，足以支撑完整训练
- 需要对模型架构进行重大修改

**选择迁移学习**：
- 数据量有限（几百到几千张）
- 任务与预训练模型的训练任务相关（如 ImageNet 预训练用于自定义图像分类）
- 希望快速迭代、节省训练成本
- 需要更好的模型鲁棒性和泛化能力

### 预训练模型的效果优势
研究表明，预训练模型可以显著提高模型在下述场景中的表现：
- 噪声数据（标签损坏）
- 类别不均衡
- 对抗扰动
- 分布外数据（OOD）

**总结**：YOLO11 是目前 YOLO 系列的最新版本，由 Ultralytics 公司开发和维护。在实际应用中，除非有特殊需求或海量数据，否则建议使用迁移学习（基于预训练模型微调），可以大幅节省训练成本并取得更好的效果。
